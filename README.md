# Verse App

<img width="515" height="955" alt="Screenshot 2025-07-24 110305" src="https://github.com/user-attachments/assets/55a54ab8-c2c6-43e4-8d20-206e1dfa2099" />

<img width="417" height="954" alt="Screenshot 2025-07-24 110400" src="https://github.com/user-attachments/assets/faf6e390-f2f9-4d9d-aa79-59c8c6fadded" />

This is my current ongoing project. It is a mobile-first Bible verse memorization app. Users can create, save, and organize collections of Bible verses or Bible reading plans. Users can search for verses by reference or keyword(s) using semantic search. Across the whole app are passage bottom sheets, which contain verses in their original formatting, cross references, and other data such as the next practice due date, and the number of total users who saved the verse or practiced it. Users can publish their collection or reading plan to a community page where other users can save it to their account. There is a main search feature where users can quick search for a passage, public collections, reading plans, or other users. Users can add friends and see friends' profiles, verses, stats, etc. There is a central notification system that notifies you of your freind's activity or of trending content in the public community page.

Tech Stack: I am learning a lot about real-world systems as this will have users. From authentication & authorization to unit & integration testing, and optimizing SQL queries and separating read and write operations for performance. Front-end is React Native Expo, Back-end is ASP.NET Minimal API with Postgres database.
# Word Riddle

<img width="1558" height="652" alt="image" src="https://github.com/user-attachments/assets/8d64815c-7e6c-45f9-9b76-647559264184" />

A Wordle clone written in C# and XAML as an Avalonia application (for cross-platform). This was the final project for my college class ".NET Programming". I used an Oracle database to have a leaderboard.
This was also my first time making an API and figuring out how to set up a database connected to the API and where to host in in the real-world.
This project also exercised my logical thinking figuring out how to make the core Wordle game to work just like the original.
Client-side repository: https://github.com/Joshrobertson023/WordRiddleFresh
API repository: https://github.com/Joshrobertson023/WordRiddleAPI
